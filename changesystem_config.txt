问题：工价系数 1.56 硬编码 → 配置化
建议方案
1. 数据库层面
在 system_config 表中新增配置项：
INSERT INTO system_config (key, value, description, category) VALUES
('process_coefficient', '1.56', '工价系数', 'calculation'),
('process_coefficient_editable', 'true', '工价系数是否可按客户/型号自定义', 'calculation');
2. 代码层面
修改 backend/utils/costCalculator.js：
constructor(config = {}) {
    this.overheadRate = config.overheadRate || 0.2;
    this.vatRate = config.vatRate || 0.13;
    this.insuranceRate = config.insuranceRate || 0.003;
    this.exchangeRate = config.exchangeRate || 7.2;
    this.processCoefficient = config.processCoefficient || 1.56; // 新增
    this.profitTiers = config.profitTiers || [0.05, 0.10, 0.25, 0.50];
}

calculateBaseCost({ materialTotal, processTotal, packagingTotal, freightCost, includeFreight = true }) {
    // 使用配置的系数，而不是硬编码
    const adjustedProcessTotal = processTotal * this.processCoefficient;
    // ...
}
3. 前端配置界面
在系统配置页面新增"计算参数配置"模块：
系统配置
├── 基础配置
│   ├── 管销率: 20%
│   ├── 增值税率: 13%
│   ├── 保险率: 0.3%
│   ├── 工价系数: 1.56  ← 新增，可编辑
│   └── 汇率: 7.2
└── 利润区间配置
    └── [5%, 10%, 25%, 50%]
4. 扩展建议（可选）
如果未来需要更灵活的配置（如不同型号不同系数），可以：
在 models 表中新增 process_coefficient 字段
在 customers 表中新增 process_coefficient 字段
优先级：客户配置 > 型号配置 > 系统默认配置
