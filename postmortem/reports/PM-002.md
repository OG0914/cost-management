# PM-002: 图片预览层级(z-index)问题

## 元信息
- **ID**: PM-002
- **修复Commit**: 40bf90c
- **影响范围**: 前端型号管理页面
- **根因类别**: `LOGIC`

## 问题描述
型号管理页面的图片预览功能无法正常显示，预览层被其他元素遮挡。

## 根因分析
- **直接原因**: Element Plus 的 el-image 组件预览功能默认 z-index 不够高
- **深层原因**: 
  1. 页面中存在多个定位元素，z-index 层叠关系混乱
  2. 未使用 `preview-teleported` 将预览层传送到 body
- **涉及文件**: `frontend/src/views/model/ModelManage.vue`

## 触发条件 ⚠️ 关键
```yaml
patterns:
  - type: code_pattern
    regex: "<el-image[^>]*(?!preview-teleported)[^>]*preview"
    description: "el-image 组件启用预览但未设置 preview-teleported"
  - type: code_pattern
    regex: "z-index:\\s*(?:auto|\\d{1,2}(?!\\d))"
    description: "z-index 值过低可能导致层级问题"
  - type: keyword
    words: ["el-image", "preview", "z-index", "层级", "遮挡", "预览"]
```

## 修复方案
```vue
<el-image
  :preview-teleported="true"
  :z-index="9999"
  ...
/>
```

## 预防检查清单
- [ ] 所有 el-image 预览功能必须添加 `preview-teleported`
- [ ] Modal/Dialog 中的图片预览需设置更高 z-index
- [ ] 新增预览功能后必须测试弹出层级
