description: 自动总结更改并等待满意确认后才会自动提交并推送到云端
---

为了保证您对代码更改拥有绝对的控制权并在确认后享受自动化的顺滑，我们在执行此工作流时必须严格遵守以下流转规则：

1. **审视并与老板确认改动**：请使用 `git status` 及了解最近修改的内容。除非在本轮对话或此前的交流中老板已经明确知晓并认可了代码变动，否则主动列出主要改动，让老板确认“当前代码修改得合理”。
2. **起草总结信息**：根据确定的修改内容，提炼并生成一行精炼的单行中文 commit message（遵循规范格式如 `[feat] 详情组件: 修改发货卡片逻辑`）。
3. **等待最终命令**：必须在此步骤停下来，向老板请示：“您看这句提交摘要满意吗？如果满意，我将为您启动一键自动推送流程。” —— 直到听到老板给出的正面确认（如“满意”、“可以”、“继续”、“上”），才允许往后执行。若是要求修改总结，则需修改总结后重新确认。
// turbo
4. 当老板确认无误后，全自动（静默但展示行为给老板）执行 `git add .`。
// turbo
5. 使用老板最终认可的那句摘要，全自动执行 `git commit -m "<在此替换为确认的提交摘要>"`。
// turbo
6. 全自动执行 `git push origin HEAD` 将确认完毕的代码推送到远程仓库。
7. 在所有步骤成功完成后，向老板简明地汇报推送顺利结束。
