# 成本管理系统 Agent Team

针对成本管理系统完善阶段的 **6人混合Agent Team**。

## 团队成员

| Agent | 核心职责 | 触发关键词 |
|-------|---------|-----------|
| [成本计算审计师](./cost-auditor.md) | 验证成本计算逻辑、报价单价格链 | 成本计算、报价单、管销价、利润价 |
| [前端现代化专家](./frontend-modernizer.md) | 前端页面/样式现代化、组件规范 | 前端、Vue、页面、样式、UI、组件 |
| [代码质量审查员](./code-quality-reviewer.md) | 代码规范、冗余识别、重构建议 | 重构、代码质量、冗余、规范、审查 |
| [API审查师](./api-reviewer.md) | REST规范、安全性、性能优化 | API、接口、REST、路由、后端 |
| [Bug修复专家](./bug-fixer.md) | 根因分析、修复实施、回归预防 | Bug、修复、错误、异常、问题 |
| [测试验证员](./test-verifier.md) | 功能验证、Playwright测试 | 测试、验证、验收、Playwright |

## 使用方法

### 方式1：手动指定Agent
在对话中明确指定Agent名称：
```
@成本计算审计师 帮我检查一下这个报价单计算逻辑
@前端现代化专家 优化一下这个按钮页面的样式
@代码质量审查员 审查一下这个组件的代码质量
```

### 方式2：关键词自动触发
系统会根据您的描述自动匹配最合适的Agent：
- 提到"成本计算" → 自动调用成本计算审计师
- 提到"前端页面" → 自动调用前端现代化专家
- 提到"Bug修复" → 自动调用Bug修复专家

### 方式3：多Agent协作
复杂任务可以召唤多个Agent并行工作：
```
请 @成本计算审计师 和 @API审查师 一起审查这个功能实现
```
