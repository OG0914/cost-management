# 多币种扩展性分析

## 一、当前系统币种使用情况

### 1.1 已实现的币种支持

当前系统已经具备基础的多币种架构：

**数据库层面：**
- `materials` 表有 `currency` 字段（默认 'CNY'）
- `packaging` 表有 `currency` 字段（默认 'CNY'）
- `quotations` 表有 `currency` 字段（默认 'CNY'）

**前端界面：**
- 材料管理页面支持选择币种：CNY（人民币）、USD（美元）、EUR（欧元）
- 包材管理页面同样支持多币种选择

**系统配置：**
- `exchange_rate`：通用汇率（CNY/USD），默认 7.2
- `fob_shenzhen_exchange_rate`：FOB深圳运费专用汇率，默认 7.1

### 1.2 当前的局限性

虽然数据库和前端已支持多币种，但存在以下问题：

1. **汇率配置单一**：只配置了一个通用汇率，无法区分不同币种
2. **转换逻辑缺失**：没有统一的币种转换服务
3. **计算器硬编码**：成本计算时可能假设所有价格都是人民币
4. **报价单币种**：报价单只能以单一币种输出
