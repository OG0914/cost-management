# 代码审查清单

## Instructions

### 1. 类型安全检查

- [ ] 无 `any` 类型使用
- [ ] 无 `@ts-ignore` 注释
- [ ] 函数参数和返回值有明确类型
- [ ] Props 接口定义完整

### 2. 安全检查

- [ ] 无硬编码密钥或密码
- [ ] 敏感数据不打印到日志
- [ ] SQL 使用参数化查询（Prisma ORM）
- [ ] 用户输入有验证

### 3. 性能检查

- [ ] 无 N+1 查询问题
- [ ] 大列表有分页
- [ ] React 组件适当使用 memo
- [ ] 无不必要的 useEffect 依赖

### 4. 代码风格

- [ ] 函数不超过 50 行
- [ ] 组件不超过 200 行
- [ ] 无 console.log（使用 Logger）
- [ ] 注释简洁在代码右侧

### 5. 错误处理

- [ ] try-catch 包裹异步操作
- [ ] 错误有适当的用户提示
- [ ] 不吞掉异常（空 catch）

## 输出格式

```
Summary: <一句话总结>
Findings:
- <问题1>
- <问题2>
Follow-up:
- <建议修复1>
```
