# 命令解释器

## 触发条件
**始终生效** - 在执行以下任何操作之前必须先解释：
- 终端命令（npm, git, pip, node 等）
- 文件创建/修改/删除
- 配置变更
- 依赖安装
- 构建/部署操作

## 执行流程
1. 识别即将执行的操作类型
2. 用简洁的中文描述该操作的原理
3. 说明执行后的预期效果
4. 然后再执行操作

## 输出格式

### 模板
```
**原理及作用**：[简洁的中文解释，说明这个操作做什么、为什么要做]

[然后执行实际操作]
```

### 示例

#### 终端命令
```
**原理及作用**：`npm run build` 会调用 Vite 打包工具，将 Vue 源码编译为生产环境的静态文件，输出到 dist 目录。用于验证代码是否能正确构建。
```

#### 文件修改
```
**原理及作用**：修改 `CostAdd.vue` 的表单验证逻辑，将错误提示改为红色样式，让用户更直观地看到验证失败的原因。
```

#### 依赖安装
```
**原理及作用**：`npm install axios` 会安装 HTTP 请求库 axios，用于在前端与后端 API 进行数据通信。
```

## 规范要求
- 解释必须简洁，控制在 1-2 句话
- 使用用户能理解的语言，避免过于技术化
- 说明"做什么"和"为什么"
- 对于复杂操作，可额外说明风险或注意事项
